|  |

| Description |
| **[POST] https://sa2.esmplus.com/shipping/v1/Delivery/ClaimRelease** |
| 미수령신고 주문데이터에 대해 미수령신고 철회 신청을 합니다. 철회시 바로 해제되는 것은 아니며 고객에게 안내 후 이의 없을 경우 배송완료일 + 7일차 자동으로 미수령신고 해제됩니다. 미수령신고 철회되어야 정산되며 고객이 또 받지 못할 경우 양사 CS를 통해 처리필요합니다 CS가 강성이 될 수 있으므로 반드시 고객이 실제 물건을 받았는지를 확인한 후 미수령신고철회처리를 요청합니다. |

### **Request Description**

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| **항목** | **항목명** | **필수여부** | **Type** | **Description** |
|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| orderNo | 주문번호 | Y | long | 철회하고자 하는 주문번호 입력 |
| ClaimCancelType | 조회기준구분 | Y | int | 원송장에서 송장번호가 잘 못 되어 미수령신고가 들어온 경우 송장번호 재입력타입으로 정상 송장을 재연동하며, 원송장이 이슈 없고 고객 커뮤니케이션이 끝났을 경우 2번타입을 통해 철회신청  1 : 송장번호 재입력 2 : 철회요청 메시지입력 |
| InvoiceNo | 송장번호 | Y | string | ClaimCancelType > 1일 경우 필수 |
| DeliveryCompCode | 택배사코드 | Y | int | ClaimCancelType > 1일 경우 필수  택배사코드 참고(<https://etapi.gmarket.com/142>) |
| CancelComment | 철회요청메시지 | Y | string | ClaimCancelType > 2일 경우 필수 고객에게 노출되는 메시지로 최대 125자까지만 노출됨 |

```
JSON
{
  "OrderNo":0,
  "ClaimCancelType":0,
  "DeliveryCompCode":0,
  "DeliveryCompName":"string",
  "InvoiceNo":"string",
  "CancelComment":"string"
}
```
