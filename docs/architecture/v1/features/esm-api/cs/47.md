# 판매자문의 조회 API

> 출처: https://etapi.gmarket.com/47
> 섹션: CS API
> Tistory post id: `47`
> 추출 시점: 2026-05-29

---

|  |
| --- |
| Description |
| [POST] https://sa2.esmplus.com/item/v1/communications/customer/bulletin-board |
| 고객이 상품 및 주문 관련 문의한 내용을 조회합니다. 제휴사이트 (SSG.COM) 문의는 G마켓 문의글 조회시 같이 조회되며 답변 후 재답변 하더라도 답변 수정이 불가합니다. 또한 답변 내용은 1,000byte 이내로 입력할 수 있습니다. |

### **Request Description**

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| **항목** | **항목명** | **필수여부** | **Type** | **Description** |
|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| qnaType | 조회구분 | Y | int | 조회 구분  1 : 옥션(일반) 2 : 옥션(비밀글) 3 : 전체(G마켓은 해당 구분만 사용) |
| status | 처리상태 | Y | int | 문의 답변 처리상태 관련 조회  1 : 전체 2 : 미처리 3 : 처리완료 4 : 처리중(G마켓용) 5 : 중복문의(G마켓용) |
| type | 조회기준 구분 | Y | int | 조회 기준 구분(접수일로만 지원)  1 : 접수일 |
| startDate | 조회기준 시작일 | Y | date | - YYYY-MM-DD 형식으로 입력 - 7일 단위로 조회 가능 |
| endDate | 조회기준 종료일 | Y | date | - YYYY-MM-DD 형식으로 입력 - 7일 단위로 조회 가능 - 당일 조회 시 시작일 +1일로 조회 |

```
JSON
{
  "status": int,
  "qnaType": int,
  "type": int,
  "startDate": "2019-04-15",
  "endDate": "2019-04-15"
}
```

### **Response Description**

|  |  |  |  |
| --- | --- | --- | --- |
| **항목** | **항목명** | **Type** | **Description** |
| resultCode | 결과코드 | int | 실패했을 경우 에러코드 내려감 |
| message | 메시지 | string | 실패사유 |
| token | 토큰 | string | 답변 작성 시 문의번호와 token 함께 요청되어야 함 |
| QnaType | 사이트 구분 | int | 조회된 게시판  1 : 옥션(일반) 2 : 옥션(비밀글) 3 : 전체 |
| SellerId | 판매자 사이트ID | string |  |
| contractType | 문의유형 | string | [옥션] 전체, 상품(성능/사이즈), 배송, 교환, 반품/취소/환불, 기타  [G마켓] 상품, 배송, 반품/환불, 교환, 취소, 안전거래, 기타 |
| InformStatus | 문의답변상태 | string | 문의 사항에 대해 답변 처리됐는지 여부 미처리, 처리완료 |
| ReAsking | 재질문여부 | Boolean | 옥션/G마켓 기준 재질문 여부 체크  true : 재질문 false : 재질문 아님 |
| MessageNo | 문의번호 | string | 문의번호 기준으로 답변 처리 필요 중복문의 동일 건 문의에 대해 No별 관리 필요 |
| answer | 답변내용 | string | 판매자 답변 내용 |
| ReceiveDate | 접수일시 | date | - 판매자문의 발생 시간 - YYYY-MM-DD hh:mm:ss+09:00 형식으로 조회됨 |
| AnswerDate | 처리일시 | date | - 판매자문의 답변처리 시간 - YYYY-MM-DD hh:mm:ss+09:00 형식으로 조회됨 |
| GoodsNo | 마스터 상품번호 | string | 2.0 상품일 경우 ESM 마스터 상품번호 조회됨 |
| SiteGoodsNo | Site 상품번호 | string | Site 상품번호 조회됨 |
| PayNo | 장바구니번호 | string | 대표 장바구니번호 조회됨 |
| OrderNo | 주문번호 | string | - 대표 주문번호 조회됨 - 데이터 없을 경우, 옥션 "0" / G마켓 "-"로 내려감 |
| orderType | 주문종류 | string | 참고용 데이터로, 아래 텍스트로 내려감 일반주문, 물류주문, 상품문의는 null |
| SellStatus | 판매진행상태 | string | 참고용 데이터(현재 옥션만 지원) 접수 시점 해당 주문의 최종 주문상태 예) 교환물품도착, 반품보류 등 |
| Title | 제목 | string | 문의 제목 |
| Details | 문의내용 | string | 문의 내용 옥션은 채팅 형으로 판매자가 답변하기 이전 문의를 묶어서 채팅 순으로 조회됨 판매자가 답변 후 재문의한 경우 "[처리완료내역][문의번호:최신문의번호]History내용" 으로 기존 문의한 내용이 내려감(판매자 답변 내용은 내려가지 않으므로 문의번호로 조회 필요) 미처리된 내용은 [미처리][문의번호:]로 내려감 |
| InquirerName | 문의자명 | string | 문의자 이름 |
| InquirerPhone | 문의자 연락처 | string | 문의자 전화번호 |
| branchId | 지점코드 | string | 당배관 판매자위함(추후 제공) |

### **Error Code**

|  |  |  |
| --- | --- | --- |
| **결과코드** | **메시지** | **원인** |
|  |  |  |
| --- | --- | --- |
| 1000 | 조회된 기간에 조회 대상이 없습니다 | 기간 내 문의건이 없을 경우 |
| 1000 | 7일 기간이상 조회 요청이 불가능합니다. | 조회기간 날짜가 7일을 초과할 경우 |
