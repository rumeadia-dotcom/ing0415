# 상품 승인 요청

**API 적용 가능한 구매자 사용자 지역: 한국，대만**

임시저장 상태의 상품은 승인요청->승인완료 단계를 거처 상품 페이지에 노출됩니다.

주의: "인벤토리(XXXX)는 승인요청 상태가 아닙니다." 메시지는 "업체상품[XXXX]이 승인요청 가능한 상태가 아닙니다." 의 의미입니다.

'임시저장' 상태에서만 '승인요청'이 가능합니다.

상품생성, 수정 API 에서 requested 파라메터를  true 로 입력할 경우 자동으로 판매 승인요청이 진행됩니다.

## Path

PUT

/v2/providers/seller\_api/apis/api/v1/marketplace/seller-products/{sellerProductId}/approvals

### Example Endpoint

https://api-gateway.coupang.com/v2/providers/seller\_api/apis/api/v1/marketplace/seller-products/{sellerProductId}/approvals

## Request Parameters

### Path Segment Parameter

| Name | | | | | Required | Type | Description |
| --- | --- | --- | --- | --- | --- | --- | --- |
| sellerProductId | | | | | O | Number | 등록상품ID  상품 생성 시, 결과 값으로 획득한 ID |

### Request Example

not require body

## Response Message

| Name | | | | | Type | Description |
| --- | --- | --- | --- | --- | --- | --- |
| code | | | | | String | 결과코드  SUCCES/ERROR |
| message | | | | | String | 메세지 |
| data | | | | | String | 등록상품ID  승인요청한 등록상품ID |

### Response Example

```
{
  "code": "SUCCESS",
  "message": "1320***567 승인 요청되었습니다. 승인 내용에 문제가 있을 경우 승인 요청이 처리되지 않을 수 있습니다.",
  "data": "1320***567"
}
```

### Error Spec

| HTTP 상태 코드 (오류 유형) | 오류 메시지 | 해결 방법 |
| --- | --- | --- |
| 400 (요청변수확인) | "임시저장", "승인완료", "승인반려", "부분승인완료" 상태의 상품만 승인 요청 가능합니다. | 승인요청하는 등록상품의 상태가 올바른지 재확인합니다. |
| 400 (요청변수확인) | 상품 정보가 등록 또는 수정되고 있습니다. 잠시 후 다시 조회해 주시기 바랍니다. [1320\*\*\*567] | 최소 10분 이후에 다시 상품승인요청을 수행합니다. |

APPROVE\_PRODUCT
