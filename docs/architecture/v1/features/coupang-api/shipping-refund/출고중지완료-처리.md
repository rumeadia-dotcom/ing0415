# 출고중지완료 처리

> 출처: <https://developers.coupangcorp.com/hc/ko/articles/360034320673>
> Zendesk article id: `360034320673`
> 섹션: 배송 / 환불 APIs
> 추출 시점: 2026-05-28

---

**API** **적용** **가능한** **구매자** **사용자** **지역****:** **한국**

고객이 주문을 취소처리 하여

반품요청목록 조회 시 상태가 출고중지요청(**RELEASE\_STOP\_UNCHECKED**) 이거나 반품접수미확인(**RETURNS\_UNCHECKED**) 이면서 releaseStatus가 "N" 일 경우에 사용가능합니다.   
아직 상품을 발송하지 않아, 고객 요청대로 상품을 출고하지 않았을 때 사용합니다.

<주의!> 출고 중지 요청 상품이 배송되면 왕복 반품 배송비는 판매자의 귀책입니다.

## Path

PATCH PUT

/v2/providers/openapi/apis/api/v4/vendors/{vendorId}/returnRequests/{receiptId}/stoppedShipment

### Example Endpoint

https://api-gateway.coupang.com/v2/providers/openapi/apis/api/v4/vendors/A00012345/returnRequests/363667/stoppedShipment

### Error Spec

| HTTP 상태 코드(오류 유형) | 오류 메시지 | 해결 방법 |
| --- | --- | --- |
| 400 (요청변수확인) | 이미 반품이 완료 되었습니다. | 반품요청 단건조회 API로 접수번호(receiptId)의 반품상태가 완료인지 확인합니다. 반품완료된 경우 더 이상 출고중지 완료 처리하지 않도록 조치합니다. |
| 400 (요청변수확인) | 취소반품접수수량과 cancelCount가 일치하지 않습니다. | 출고중지할 수량(cancelCount)을 반품접수수량과 동일하게 입력했는지 확인합니다. |
| 400 (요청변수확인) | 상품준비 중 또는 배송지시 상태가 아닙니다. 배송완료 | 출고중지할 주문이 상품준비중 또는 배송지시 상태인지 확인합니다. 배송완료 상태일 경우 처리 시 에러가 발생합니다. |
| 400 (요청변수확인) | cancelType이 A 또는 RA가 아닙니다. | 접수번호(receiptId)가 올바른 값인지 올바른 반품상태 인지 확인하고 더 이상 출고중지 완료 처리를 하지 않도록 조치합니다. |
| 500 (요청변수확인) | 출고 되어 처리할 수 없습니다. | 반품요청 단건조회 API로 접수번호(receiptId)의 반품상태가 완료인지 확인합니다. 반품완료된 경우 더 이상 출고중지 완료 처리하지 않도록 조치합니다. |

### URL API Name
