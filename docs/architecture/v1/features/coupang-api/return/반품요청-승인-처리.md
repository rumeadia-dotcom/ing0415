# 반품요청 승인 처리

> 출처: <https://developers.coupangcorp.com/hc/ko/articles/360034562513>
> Zendesk article id: `360034562513`
> 섹션: 반품 APIs
> 추출 시점: 2026-05-28

---

**API** **적용** **가능한** **구매자** **사용자** **지역****:** **한국****,** **대만**

반품요청 목록조회 시 receiptStatus가 입고완료(VENDOR\_WAREHOUSE\_CONFIRM)인 반품 건에 대해 처리 가능합니다.

[반품 상품 입고 확인 처리] 후, [반품 요청 승인 처리]를 진행하시면 환불 처리됩니다.   
선환불(빠른환불) 정책에 의해 환불처리되거나, 입고확인이 완료된 반품 건의 경우 일정 시간 경과 후 쿠팡시스템에 의해 반품이 승인 처리됩니다. 이 경우에는 해당 API 호출이 불필요합니다.

\*[반품 API Workflow](https://developers.coupangcorp.com/hc/ko/articles/360033643294)를 확인하시면, 보다 쉽게 이해하실 수 있습니다.

## Path

PATCH PUT

/v2/providers/openapi/apis/api/v4/vendors/{vendorId}/returnRequests/{receiptId}/approval

### Example Endpoint

https://api-gateway.coupang.com/v2/providers/openapi/apis/api/v4/vendors/A00013264/returnRequests/363585/approval

## Request Parameters

| vendorId | | | | | O |  | 업체코드 |
| receiptId | | | | | O |  | 반품접수번호  ReceiptId는 반품요청 목록 조회API를 통해 확인가능합니다.  ReceiptId는 반드시 number타입이어야 합니다. |

| cancelCount | | | | | O | Number | 반품접수 수량 |

### Error Spec

| HTTP 상태 코드 (오류 유형) | 오류 메시지 | 해결 방법 |
| --- | --- | --- |
| 400 (요청변수확인) | 취소반품접수내역이 존재하지 않습니다. | 접수된 반품내역이 고객 또는 상담사에 의해 철회되었거나 잘못된 접수번호를 입력할 경우 발생합니다. 반품철회 이력 API로 반품철회이력이 있는지 확인합니다. |
| 400 (반품상태확인) | 이미 반품이 완료 되었습니다. | 반품요청 단건조회 API로 접수번호(receiptId)의 반품상태가 완료인지 확인합니다. 빠른환불 또는 회수 트래킹 정보에 의해 반품완료된 경우 더 이상 중복 승인처리하지 않도록 조치합니다. |
| 400 (요청변수확인) | 취소반품접수수량과 cancelCount가 일치하지 않습니다. | 올바른 반품접수수량을 입력했는지 확인합니다. 접수된 반품접수 수량과 입력한 cancelCount 수량이 일치하지 않을 경우 에러가 발생합니다. |
| 500 (반품상태확인) | 환불 승인(대기)/접수철회 요청된 쿠폰의 상태가 승인 할 수 없는 상태이거나 수량이 맞지 않습니다. | 반품요청 단건조회 API로 접수번호(receiptId)의 반품상태가 완료인지 확인합니다. 빠른환불 또는 회수 트래킹 정보에 의해 반품완료된 경우 더 이상 중복 승인처리하지 않도록 조치합니다. |
