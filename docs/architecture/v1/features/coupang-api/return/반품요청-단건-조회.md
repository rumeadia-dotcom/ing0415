# 반품요청 단건 조회

> 출처: <https://developers.coupangcorp.com/hc/ko/articles/360034562353>
> Zendesk article id: `360034562353`
> 섹션: 반품 APIs
> 추출 시점: 2026-05-28

---

**API** **적용** **가능한** **구매자** **사용자** **지역****:** **한국****,** **대만**

반품 접수번호를 이용하여 반품 단건 조회를 수행합니다.

## Path

GET

/v2/providers/openapi/apis/api/v6/vendors/{vendorId}/returnRequests/{receiptId}

### Example Endpoint

https://api-gateway.coupang.com/v2/providers/openapi/apis/api/v6/vendors/A00012697/returnRequests/363585

## Request Parameters

### Path Segment Parameter

| Name | | | | | Required | Type | Description |
| --- | --- | --- | --- | --- | --- | --- | --- |
| vendorId | | | | | O | String | 판매자 ID |
| receiptId | | | | | O | Number | 반품접수번호 (취소번호는 지원되지 않습니다)  ReceiptId는 반품요청 목록 조회API를 통해 확인가능합니다.  ReceiptId는 반드시 number타입이어야 합니다. |

### Request Example

not require body

### Error Spec

| HTTP 상태 코드(오류 유형) | 오류 메시지 | 해결 방법 |
| --- | --- | --- |
| 400 (요청변수확인) | ReceiptId doesn't belong to the vendorId, or cannot find the Receipt by the given id | 조회하는 반품접수번호(receiptId)가 고객에 의해 철회 되었는지 확인합니다. [반품철회 이력 접수번호로 조회 API](https://developers.coupangcorp.com/hc/ko/articles/360034027374)로 철회내역을 조회할 수 있습니다. 철회된 반품접수 번호는 더이상 조회하지 않도록 처리합니다. |

### URL API Name

GET\_RETURN\_REQUEST\_BY\_RECEIPT
