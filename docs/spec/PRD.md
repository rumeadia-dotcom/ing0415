## 1. 프로젝트 개요
---
### 한 줄 정의
다중 마켓 상품 자동 등록 서비스

### 제품 목표
개인 판매자가 네이버 스마트스토어, 쿠팡, 지마켓 등 여러 온라인 마켓 판매 채널에 상품을 수동으로 등록하는 데 소모되는 시간과 노력을 자동화를 통해 획기적으로 절감하고, 상품 등록 과정을 간소화하여 판매 효율을 극대화하는 것을 목표로 합니다.

### 배경
오늘날 개인 판매자들은 경쟁 심화로 인해 더 많은 판매 채널에 상품을 노출시켜야 하는 필요성을 느낍니다. 하지만 각 마켓별로 상이한 상품 등록 양식과 복잡한 절차 때문에 많은 시간을 수동 등록에 할애하고 있습니다. 이는 판매자들의 생산성을 저해하고 신속한 시장 대응을 어렵게 만듭니다. 이러한 문제를 해결하고 판매자들이 핵심 비즈니스에 집중할 수 있도록 상품 등록 과정을 자동화하는 웹 서비스의 필요성이 커지고 있습니다.

### 사용자 문제
개인 판매자들이 다수의 온라인 마켓에 동일한 상품을 등록할 때, 각 마켓별로 반복적으로 정보를 입력하고 이미지를 업로드하는 데 많은 시간과 노동력이 소모됩니다.

### 해결 방안
판매자가 상품 정보를 한 번만 입력하면 연결된 여러 마켓에 자동으로 일괄 등록해주는 기능을 제공하며, 통합 로그인 시스템을 통해 여러 마켓 계정을 한 곳에서 편리하게 관리할 수 있도록 합니다.

### 차별점
다수의 마켓 플레이스 상품 등록 과정을 한 곳에서 통합 관리하고 자동화하여 판매자의 시간과 노력을 절감하는 데 초점을 맞춥니다. 또한, 향후 웹뿐만 아니라 앱, 데스크탑 애플리케이션 등 다양한 기기에서 접근 가능하도록 확장성을 고려하여, 언제 어디서든 편리하게 상품을 관리하고 등록할 수 있는 유연한 환경을 제공합니다.

### 타겟 사용자
네이버 스마트스토어, 쿠팡, 지마켓 등 2개 이상의 온라인 마켓 플레이스에서 상품을 판매하는 개인 판매자. 특히, 신규 상품을 자주 등록하거나 다양한 채널 확장을 계획하고 있으나 상품 등록 업무에 많은 시간을 할애하는 소규모 비즈니스 운영자 및 1인 기업가.

### 사용자 시나리오
개인 판매자가 신상품을 여러 온라인 마켓에 동시에 등록해야 할 때, 이 서비스에 접속하여 한 번 로그인합니다. 그 후, 새 상품 정보를 양식에 맞춰 입력하고, 미리 연결해둔 마켓 계정 중 상품을 등록하고자 하는 마켓들을 선택합니다. '일괄 등록' 버튼을 누르면, 시스템이 각 마켓별 요구사항에 맞춰 상품 정보를 자동으로 변환하여 선택된 모든 마켓에 상품을 등록합니다. 등록이 완료되면 대시보드에서 각 마켓별 등록 성공 여부와 오류 내용을 한눈에 확인합니다.

### 핵심 지표
월간 총 상품 등록 건수, 월간 활성 사용자 수(MAU), 평균 상품 등록 시간 단축률(%) 측정, 사용자 피드백을 통한 만족도 점수(NPS) 향상

### 리스크
각 마켓 플레이스의 API 정책 변경에 대한 신속한 대응 능력, 사용자 상품 정보 및 마켓 계정 정보의 보안 취약점 발생 가능성, 기존 시장에 존재하는 유사 서비스와의 경쟁 심화 및 차별화 전략 부재.

### 카테고리
> 생산성 / 업무

### 사용자 역할
> 판매자

### 디바이스 (기기)
> Web, App, WEBAPP

## 1. 다중 마켓 상품 일괄 등록
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

사용자가 한 번의 상품 정보 입력으로 여러 온라인 마켓(예: 네이버 스마트스토어, 11번가, 지마켓, 옥션, 쿠팡)에 상품을 동시에 등록할 수 있도록 합니다.

**수용 기준:**
```
1. ○ 사용자는 상품 정보를 한 번 입력하여 최소 3개 이상의 지정된 마켓에 동시 등록할 수 있다.
2. ○ 각 마켓별 요구사항에 맞춰 상품 정보(상품명, 가격, 카테고리, 상세 설명, 이미지 등)가 자동으로 매핑되어 등록된다.
3. ○ 등록 과정 중 발생하는 오류는 사용자에게 명확하게 안내되며, 오류 발생 마켓을 식별할 수 있다.
```

## 2. 통합 로그인 및 마켓 계정 관리
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

사용자가 서비스에 한 번 로그인하여 연결된 모든 마켓 계정을 안전하게 관리하고, 상품 등록에 활용할 수 있도록 합니다.

**수용 기준:**
```
1. ○ 사용자는 서비스에 로그인 후 네이버 스마트스토어, 쿠팡, 지마켓 등 여러 마켓 계정을 연결하고 관리할 수 있다.
2. ○ 연결된 마켓 계정 정보를 암호화하여 안전하게 저장하고, 상품 등록 시 자동으로 해당 계정으로 연동한다.
3. ○ 각 마켓의 로그인 및 API 연동 정책(예: OAuth)을 준수하여 안전하게 인증 절차를 수행한다.
```

> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: 판매자 | **기기**: Web, App, WEBAPP


#### 2.1.2 비밀번호 강도 검사 및 가이드

## 6. 다중 마켓 주문 자동 수집 및 배송 자동화

판매자가 여러 마켓에 접속해 주문을 수동으로 확인하고 운송장을 입력하는 반복 작업을 제거합니다. MarketCast 1곳에서 4마켓 주문을 통합 수집하고, 로젠택배 집하 예약·운송장번호 채번을 자동 처리한 뒤, 마켓 송장 일괄 제출까지 클릭 최소화로 끝냅니다.

**전제 조건**
- 로젠택배 B2B 계약 완료 (`userId` = 연동업체코드 / `custCd` = 거래처코드 발급).
- 운송장번호는 출력 시점(집하 전)에 `getSlipNo` API 로 즉시 채번 가능.

**수용 기준:**
```
1. ○ 4마켓(네이버 / 쿠팡 / G마켓 / 옥션) 신규 주문이 10분 주기로 자동 수집되어 통합 목록에 노출된다.
2. ○ 주문 수집 직후 로젠 집하 예약과 운송장번호 채번이 자동 실행되어 status='logen_registered' 로 전환된다.
3. ○ 판매자는 "운송장 출력" 클릭 1번으로 로젠 인쇄 팝업을 열어 라벨을 물리 출력할 수 있다.
4. ○ "송장 일괄 제출" 클릭 1번으로 4마켓에 동시 송장번호 제출이 진행되며, 마켓별 성공/실패 결과를 확인할 수 있다.
5. ○ 로젠 API 실패 시 자동 재시도 3회 후 수동 처리 다이얼로그로 유도된다.
```


### 6.1 주문 자동 수집
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 판매자 | **기기**: Web, App, WEBAPP

Supabase pg_cron 이 10분 주기로 Edge Function `orders-sync` 를 호출하여 4마켓의 `결제완료/배송대기` 주문만 수집합니다. `(market_id, external_order_id, seller_id)` unique 제약으로 중복 방지.

**수집 데이터**: 주문번호 / 수취인명 / 주소 / 연락처 / 상품명 / 수량 / 금액 / 상태.

**마켓별 주문 조회 API**

| 마켓 | API |
|---|---|
| 네이버 | `GET /external/v1/pay-order/seller/orders/new-pay-waiting` |
| 쿠팡 | `GET /v2/.../ordersheets?status=ACCEPT` |
| G마켓 | ESM `getOrderList` (주문상태: 결제완료) |
| 옥션 | 동일 ESM API (site='A') |


### 6.2 로젠 집하 예약 자동 등록
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 판매자 | **기기**: Web, App, WEBAPP

주문 수집 직후 Edge Function `logen-register-shipment` 가 자동 실행되어 (1) `getSlipNo` 로 운송장번호를 즉시 채번하고 (2) `registerOrderData` 로 주문 건별 집하 예약을 등록한 뒤 (3) `orders.waybill_number` / `orders.logen_order_id` / `orders.status='logen_registered'` 로 DB 업데이트합니다.

**실패 처리**: 재시도 3회 (지수 백오프) → 최종 실패 시 `orders.status='logen_failed'` + 알림 배너 → 수동 처리 다이얼로그 (운송장번호 직접 입력) 유도.


### 6.3 운송장 출력 (클릭 1번)
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 판매자 | **기기**: Web, App, WEBAPP

물리적 운송장 라벨 출력 단계 — 자동화 범위 내 유일한 수동 단계. `status='logen_registered'` 주문을 전체 또는 선택하여 `outSlipPrintPop` 로젠 인쇄 팝업을 열고 프린터로 출력합니다. 출력 후 확인 버튼 클릭 시 `orders.status='waybill_printed'` 로 전환. 운송장번호는 §6.2 시점에 이미 확보된 상태입니다.


### 6.4 마켓 송장 일괄 제출
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 판매자 | **기기**: Web, App, WEBAPP

"송장 일괄 제출" 버튼 1회 클릭으로 `status='waybill_printed'` 주문 전체에 대해 4마켓 송장번호 동시 제출을 수행합니다. 설정에서 "출력 후 자동 제출" ON 시 §6.3 출력 완료 직후 자동 트리거됩니다.

**마켓별 송장 제출 API**

| 마켓 | API | 인증 |
|---|---|---|
| 네이버 | `PATCH /external/v1/orders/{orderId}/dispatch` | OAuth 2.0 |
| 쿠팡 | `PUT /v2/.../orders/{orderId}/shipments` | HMAC-SHA256 |
| G마켓 | ESM `setShipInfo` (site='G') | ESM API Key |
| 옥션 | ESM `setShipInfo` (site='A') | ESM API Key |

**처리**: Edge Function `shipping-dispatch-job` 이 4마켓 fan-out 병렬 처리. 성공 시 `status='tracking_submitted'`, 실패 마켓은 오류 코드/메시지 보존 후 재시도 가능.


### 6.5 주문·배송 현황 대시보드
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 판매자 | **기기**: Web, App, WEBAPP

오늘 요약(신규 주문 / 로젠 등록 완료 / 출력 대기 / 제출 완료), 빠른 액션 버튼("운송장 출력" / "송장 일괄 제출"), 주문 목록(마켓·상태별 필터·검색), Realtime 자동 갱신을 제공합니다. pg_cron 수집·등록 완료 시 별도 새로고침 없이 카운터·목록이 갱신됩니다.

## 7. 로젠택배 API 연동 명세 (참고)

| API | 엔드포인트 | 용도 |
|---|---|---|
| 송장번호 채번 | `POST /lrm02b-edi/edi/getSlipNo` | 운송장번호 즉시 발급 |
| 주문 정보 일괄 등록 | `POST /lrm02b-edi/edi/registerOrderData` | 집하 예약 등록 |

**Base URL**
- 개발: `https://topenapi.ilogen.com`
- 운영: `https://openapi.ilogen.com`

**인증**: `userId` (연동업체코드) + `custCd` (거래처코드) — B2B 계약 시 발급, Edge Function env var 로 관리.

## 8. 주문·배송 데이터 모델 (참고)

```sql
orders
  id                uuid PK
  seller_id         uuid FK → sellers
  market_id         text
  external_order_id text
  receiver_name     text
  receiver_address  text
  receiver_phone    text
  product_name      text
  quantity          int
  order_amount      int
  status            enum (
                      collected |
                      logen_registered |
                      logen_failed |
                      waybill_printed |
                      tracking_submitted |
                      dispatch_failed
                    )
  logen_order_id    text nullable   -- fixTakeNo (registerOrderData 응답)
  waybill_number    text nullable   -- slipNo (getSlipNo 응답)
  carrier_code      text default 'LOGEN'
  collected_at      timestamptz
  logen_registered_at timestamptz nullable
  waybill_printed_at  timestamptz nullable
  dispatched_at     timestamptz nullable
  UNIQUE (market_id, external_order_id, seller_id)

shipping_jobs
  id            uuid PK
  seller_id     uuid FK
  status        enum (pending | running | partial | succeeded | failed)
  order_count   int
  success_count int
  failed_count  int
  created_at    timestamptz
  completed_at  timestamptz nullable

shipping_job_results
  id            uuid PK
  job_id        uuid FK → shipping_jobs
  order_id      uuid FK → orders
  market_id     text
  status        enum (success | failed)
  error_code    text nullable
  error_message text nullable

logen_credentials              -- 셀러별 로젠 API 자격증명 (pgcrypto 암호화)
  id            uuid PK
  seller_id     uuid FK → sellers UNIQUE
  user_id_enc   bytea          -- userId (연동업체코드) 암호화
  cust_cd_enc   bytea          -- custCd (거래처코드) 암호화
  sender_name   text
  sender_address text
  sender_phone  text
  fare_ty       text default 'C'
  dlv_fare      int  default 0
  created_at    timestamptz
  updated_at    timestamptz
```
